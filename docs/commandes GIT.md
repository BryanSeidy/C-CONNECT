# 🔁 Cycle de travail

1. Se mettre à jour
git checkout dev
git pull origin dev
2. Créer une branche
git checkout -b feature/ma-feature
3. Coder

4. Commit propre
git commit -m "feat: add login API"

5. Push
git push origin feature/ma-feature

6. Pull Request
👉 Sur GitHub :
• expliquer ce que tu as fait
• comment tester

7. Code Review
• ASSOM ou SIMO valide

8. Merge dans dev

🧼 Bonnes pratiques Git
• ✅ Commits petits et fréquents
• ✅ Messages clairs
• ❌ Jamais push sur main
• ✅ Toujours pull avant coder

✍️ Exemples de commits
• feat: add product API
• fix: correct login bug
• style: improve UI
